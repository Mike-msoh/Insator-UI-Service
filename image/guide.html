Development Guide for Insator IoT API service

▶ IoT Test Procedure

 

[1] User Registration / Approval

▶ [Step #1] User Registration

  - Log in to Insator IoT Portal and proceed with membership application.

    : Once you obtained the membership, you should receive an approval from a manager (Master or Admin) in order to log in. 

    : Enter required value, click on Confirm button to finish the membership application, and wait for the manager’s approval. 

 

▶ [Step #2] User Approval

- After requesting for user registration, you need an approval from SDS Admin, which takes 1~2 business days.

- After user approval is completed, you receive a notification via email that you registered when you applied for the membership

 

[2] Device Meta Data Registration

▶ [Step #3] Site Registration

  - Re-login using the account that you applied for in scenario #1 and got approved in scenario #2 and register the site in Business > Site Management menu. (Ex. Jamsil-01)

 

▶ [Step #4] Thing Model Type Registration

  - Register the model type in Asset > Model Type Management menu. (Ex. WallPad-01)

 

▶ [Step #5] Thing Model Registration

  - Select the model type you registered in scenario #4 and register the model in Asset > Model Management menu.

    : Enter the model name (Ex. WallPadModel-01)

    : Enter the property group (Ex.WallPadAttr-01)

    : Enter the property (Ex. battery (int type) )

 

▶ [Step #6] Root Thing Registration

  - Select the site, model type and model name you registered in scenario #3, #4 and #5 in Thing > Thing Management menu, and

    : Enter the unique thing number. (Ex. WallPadSN-01)

    : Check if the thing name is automatically entered. (Ex.WallpadModel-01.WallPadSN-01)

    : Select the module name (Ex. (FST_MQTT) MQTT)

      ※ If you select FST_MQTT method, you will connect the thing using SDK emulator, and if you select FST_WEBS method, you will connect the thing using WEB emulator.

    : Select authentication method (Ex. DH_BASE _AUTH)

         . Credentials: Select either ISSUE or FIXED.

  - Check the thing that you registered on the list.

    : The first thing you registered in the portal is Root Thing. Its initial Activation status is provisional registration status and Connection status is Hold status.

 

▶ [Step #7] Edge Thing Registration

   ※ For your information, you can register Edge Thing in the portal or in emulator while connecting it.

   - Click on the Reg menu of the Root Thing that you have just registered in scenario #6 in Thing > Thing Management menu.

- Enter the unique number of Edge Thing. (Ex. DoorLockSN-01)

   - Check the registered Thing on the list.
    : In case of Edge Thing that you registered in the portal, initial Activation status is provisional registration status and Connection status is Hold status.

[3] Thing connection

Following scenario explains how to test the device through web emulator. Downloading SDK to actually connect the device and login based upon this will be made available from the point of Full Integration due in June.

▶ [Step #8] Root Thing Connection 

- Log in to web emulator.

- Click on Connection button and connect Root Thing.

- Check if the status of the Root Thing has been changed to Connection status in the web emulator.

 - Go to IoT Portal to check if the Connection status of the Root Thing that you registered has been changed from Hold status to Connection status.

▶ [Step #9] Thing Message Transfer Verification

- Select the Root Thing in the web emulator, modify the value in message window and click on message transfer button.

- Check if the property value of the Root Thing has been updated in IoT portal.

- Select Edge Thing in the web emulator, modify the value in message window and click on message transfer button.

- Check if the property value of the Edge Ting has been updated in IoT portal.

 

[4] Development using Insator API

To request REST API based upon TOKEN authentication, proceed with authentication using user ID/Password registered in Insator Portal and invoke REST API using the token issued.

 

Login request (issuance of API authentication token)

- Request URL: /v1.0/main/action/login

- method : POST

- Header :   (Authorization : UL) or  (X-CSRF-TOKEN : UL)

REST API Invocation 

- Request URL: /v1.0/main/action/login

- method : POST

- Header :   (Authorization : UL) or  (X-CSRF-TOKEN : UL)   

When you request for REST API, include the issued Token value in header information. (Authorizaion or X-CSRF-TOKEN)

Ex) When it is an API for looking up customer list,

- Request URI : /v1.0/customers

- method : GET  

- Header : Transfer the following sentences to key value of either Authorization or x-csrf-token.

Authorization       : UL O6Nph6Cm5+VWnVI/42DRV1e5+Vb…
or
X-CSRF-TOKEN     : UL O6Nph6Cm5+VWnVI/42DRV1e5+Vb…
